* Why async-installer?

** Live install & update

Traditional package managers block Emacs during installation.
async-installer runs every git clone, download, and byte-compile in a
background process.  You can install 50+ packages and keep editing
without interruption.

** Any Git host, commit-pinned

Every package is cloned from its Git repository with an explicit
commit, branch, or tag.  This gives you full reproducibility without
lock files — the same config always produces the same result.

Works with any Git host:
- GitHub
- GitLab
- Codeberg
- Gitea
- SourceHut
- Self-hosted instances
- Any URL that =git clone= accepts

** Dependency detection

After cloning a package, async-installer automatically parses the
=Package-Requires= header and reports any missing dependencies in the
=*Async Installer Dependencies*= buffer.

** Plain elisp — no macro DSL required

async-installer is built for users who configure Emacs with plain
=require=, =setq=, and =add-hook= instead of macro-based package
managers.

Popular managers like =use-package=, =straight.el=, =elpaca=, and
=leaf= wrap configuration in their own macro syntax.  This creates
several problems:

- *Learning overhead*: each macro system has its own keywords, timing
  semantics (=:defer=, =:after=, =:demand=, =:ensure=, etc.), and
  implicit behaviors that must be learned separately from Emacs Lisp
  itself.
- *Debugging difficulty*: when something goes wrong, you have to
  understand what the macro expanded into, not just what you wrote.
- *AI unfriendly*: LLMs can read and generate standard =(require
  'foo)= / =(setq ...)= / =(add-hook ...)= far more reliably than
  macro-specific syntax with implicit load-timing and side effects.
  A macro DSL is essentially a second language that both the user and
  the AI must learn on top of Emacs Lisp.

async-installer takes a different approach: *plain elisp is the
configuration language*.  You write normal =require=, =setq=,
=add-hook= — and async-installer only handles the install/update
plumbing in the background.

#+begin_example
;; Macro DSL (use-package):
(use-package magit
  :ensure t
  :defer t
  :bind ("C-x g" . magit-status)
  :config (setq magit-save-repository-buffers 'dontask))

;; Plain elisp (async-installer):
(async-installer-git-add
 "https://github.com/magit/magit.git"
 :commit "abc1234" :subdir "lisp" :main "magit-autoloads.el")
(with-eval-after-load 'magit
  (setq magit-save-repository-buffers 'dontask))
(global-set-key (kbd "C-x g") #'magit-status)
#+end_example

The plain elisp version is longer, but every line is standard Emacs
Lisp.  Nothing is implicit.  Any Emacs user — or AI — can read it
instantly.

* Features

| Feature | Command |
|---------+---------|
| Install from Git (async, pinned) | =async-installer-git-install-all-interactive= |
| Update Git packages | =async-installer-git-update-all-interactive= |
| Install from archive URL | =async-installer-archive-start= |
| Batch install + exit | =async-installer-git-install-all-and-exit= |
| Batch update + exit | =async-installer-git-update-all-and-exit= |
| Dependency detection | Automatic after clone |

* Requirements

- Emacs 27.1+
- [[https://melpa.org/#/async][async.el]] (the only external dependency)
- =git= in PATH

* Installation

Paste this into your =init.el=.  On first launch, async-installer
clones itself automatically — no manual =git clone= needed:

#+begin_src emacs-lisp
;; --- async-installer bootstrap ---
;; Self-installs on first run, then loads normally on subsequent starts.
(let ((dir (expand-file-name "external-packages/async-installer" user-emacs-directory)))
  (unless (file-exists-p (expand-file-name "async-installer.el" dir))
    (make-directory dir t)
    (message "Bootstrapping async-installer...")
    (call-process "git" nil nil nil "clone"
                  "https://github.com/zawatton21/async-installer.git" dir))
  (add-to-list 'load-path dir))
(require 'async-installer)
#+end_src

That's it. After the first launch, packages are cloned into
=~/.emacs.d/external-packages/=.

* Quick start

** Git packages (with version pinning)

#+begin_src emacs-lisp
;; From GitHub
(async-installer-git-add
 "https://github.com/minad/vertico.git"
 :commit "abc1234def")

;; From GitLab
(async-installer-git-add
 "https://gitlab.com/org/repo.git"
 :branch "main")

;; From Codeberg
(async-installer-git-add
 "https://codeberg.org/org/repo.git"
 :tag "v1.0")

;; With subdirectory and pre-build commands
(async-installer-git-add
 "https://github.com/org/repo.git"
 :branch "develop"
 :subdir "lisp"
 :main "repo.el"
 :pre-build '("make autoloads"))

;; Install all
(async-installer-git-install-all-interactive)

;; Later, update all
(async-installer-git-update-all-interactive)
#+end_src

** Archive packages (tar/gz/zip)

For ELPA tar files or other archives:

#+begin_src emacs-lisp
(async-installer-archive-add
 "https://elpa.gnu.org/packages/some-package-1.0.tar")
(async-installer-archive-start)
#+end_src

** "But I want package X from MELPA"

Find the package's Git repository (linked on the MELPA page) and use
the Git backend instead:

#+begin_src emacs-lisp
;; Instead of: (package-install 'diminish)
;; Do:
(async-installer-git-add
 "https://github.com/myrber/diminish.el.git"
 :commit "fbd5d846611bad828e336b25d2e131d1bc06b83d")
#+end_src

This gives you commit-pinned, fully reproducible installation.

* Batch mode (CI / fresh install)

#+begin_src sh
emacs --batch -l ~/.emacs.d/init.el \
  --eval "(async-installer-git-install-all-and-exit)"
#+end_src
